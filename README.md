# Исследование рынка стартапов

Учебный проект в рамках курса «Аналитик данных» Яндекс Практикума.

**Автор:** Ребрикова Мария

Полный цикл работы с сырыми данными о стартапах: предобработка пяти связанных датасетов, разведочный анализ и выводы для инвестора о том, какие категории стартапов и какие паттерны сделок заслуживают внимания.

## Стек

`Python` · `pandas` · `numpy` · `matplotlib` · `seaborn` · `phik`

## Данные

Пять связанных таблиц (~600 тыс. строк суммарно):

| Таблица | Строк | Суть |
| --- | --- | --- |
| `acquisition.csv` | 9 407 | Сделки по покупке компаний |
| `company_and_rounds.csv` | 217 774 | Компании + раунды финансирования в одной таблице |
| `people.csv` | 226 709 | Сотрудники стартапов |
| `education.csv` | 109 610 | Учебные заведения сотрудников |
| `degrees.csv` | 109 610 | Степени образования сотрудников |

## Что сделано

### Предобработка
- Нормализация имён колонок в `snake_case`, слияние дублирующихся `company id` / `company ID` в `company_id`.
- Приведение типов: даты → `datetime64`, идентификаторы и счётчики → `Int64` (с поддержкой NaN), категориальные поля → `category`.
- В `degrees` удалён префикс `p:` у `object_id` — это позволило стыковать таблицу с `people.id`.
- Разделение «монолитной» `company_and_rounds` на две логические таблицы:
  - `df_company` — одна строка на компанию (~196 тыс. строк по числу уникальных `company_id`);
  - `df_rounds` — одна строка на раунд финансирования.

  Массовые пропуски в исходной таблице объяснялись именно смешением карточек компаний с записями о раундах; после разделения пропуски в company-полях исчезают.

### Разведочный анализ
- **Динамика раундов финансирования по годам** — типичный (медианный) размер раунда, отфильтровано по годам с >50 раундов.
- **Полнота сведений об образовании** в зависимости от размера компании: компании разделены на 5 групп по числу сотрудников (микро / малые / средние / крупные / огромные); доля сотрудников без `education` падает с ростом размера.
- **Проверка ключа `network_username`** на пригодность для объединения — отклонён из-за разной семантики в разных таблицах и большого числа пропусков и дублей; пригоден только как дополнительный ключ.
- **Фильтрация компаний-целей анализа** (~40 тыс. компаний): есть хотя бы один раунд финансирования/инвестиций или статус `acquired`; присоединены агрегаты из `acquisition` — сколько раз компания была целью/покупателем, цена последней сделки.
- **Анализ выбросов в `funding_total`**: использована логарифмическая шкала и медиана — из-за сильной правосторонней асимметрии среднее и привычные процентили не работают. Типичный объём финансирования — $10^6$–$10^7$ долларов.
- **Компании, проданные за $0–1** при ненулевом финансировании: верхняя граница выбросов по межквартильному размаху ≈ 35.5 млн долларов (≈ 91-й перцентиль). Такие сделки помечены как аномальные.
- **Топ категорий стартапов**: по медианной цене сделки и по стандартному отклонению цены (≥10 сделок на категорию).
- **Связь числа раундов финансирования и статуса стартапа**: построено распределение среднего числа раундов по группам `operating` / `acquired` / `closed` / `ipo` / и пр.

## Главные выводы

- **Финансирование стартапов — сильно скошенное распределение.** Для анализа типичного уровня корректно использовать медиану и логарифмические преобразования; среднее искажается длинным правым хвостом.
- **Аномальные сделки за 0–1 доллар** существуют даже у компаний с миллионным финансированием — их стоит маркировать и исключать из общих агрегатов.
- **Категории с типично высокими ценами:** `manufacturing`, `biotech`, `hospitality`.
- **Категории с наибольшим разбросом цен (риском):** `public_relations`, `biotech`, `health`, `other`.
- **`biotech` попадает в оба топа** — высокая средняя цена сделки соседствует с высокой волатильностью. Интересный сегмент для крупных инвесторов, готовых к риску.
- **Приобретённые стартапы в среднем проходят меньше раундов**, чем действующие или вышедшие на IPO: покупка чаще происходит на ранних стадиях развития.

## Рекомендации для инвестора

- Опираться на медианные значения и логарифмические преобразования при оценке «нормальности» сделки.
- Сделки по нулевой/символической цене выделять отдельно и не включать в общие оценки.
- Категории с высокой медианной ценой привлекательны, но нужно трезво оценивать разброс — высокий `std` означает высокий риск.
- Анализ человеческого капитала компании возможен через `education` и `degrees`, но с поправкой на неполноту данных в малых компаниях.
